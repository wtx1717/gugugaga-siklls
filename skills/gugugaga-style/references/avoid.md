# Avoid

Use this file before final output and when editing the skill.

## Corpus and Copying

- Do not paste long source-corpus passages into generated outputs or reference files.
- Do not treat examples as required wording; they are style samples.
- Prefer reusable structures, mappings, and mechanisms over direct replication.
- If using private training notes, distill them into rules and omit raw subtitles.

## Style Failures

- Do not write generic internet nonsense without a system lens.
- Do not stack too many references in one short script.
- Do not explain every joke inside the script.
- Do not turn every sentence into a rhetorical question.
- Do not make the output only a glossary dump; it still needs a narrative turn.
- Do not end with a serious moral unless the user explicitly asks for one.

## Risky Terms

- Treat `石=屎` as an internal mapping; use direct low vulgarity only when the user asks for a rawer tone.
- `傻福` is a softened insult. Prefer using it for self-mockery or playful friend-mockery.
- Avoid direct use of disability-derived insults such as `唐`; if necessary, keep it implied or substitute safer wording.
- Keep sexual or adult-adjacent jokes implicit. Avoid explicit sexual description, especially around二次元属性梗.
- Be careful with minors, school contexts, and age-coded anime labels; do not sexualize them.
- Avoid escalating friend-mockery into harassment, threats, or targeted abuse.

## Quality Boundaries

- If the topic is ordinary, make it absurd through structure, not by randomly inserting memes.
- If the user requests a sensitive topic, reduce attack language and lean into system comedy.
- If the user asks for public documentation, keep private corpus, video names, raw OCR, and full annotations out of the public skill.
- If a phrase source is uncertain, do not present it as fact; mark it as a reusable pattern instead.
